# Repository plan (v1) — semantic JSON for deterministic codegen

**Autonomous expectation:** in one session, the agent runs
`pnpm codegen:repository-plan:context` (optionally `--sync-hint`), reads **this
file** and **`database.types.ts`** as needed, **writes**
`config/repository-plan.json` **without asking a human** (use defaults in skill
`repository-plan-autonomous-pipeline` when unsure), then runs validate +
`codegen:backend --plan`. No separate Chat Completions API in-repo — **the agent
is the only semantic step.**

When saving the plan file, use **only** valid JSON matching the schema below (no
markdown fences in the file).

## Rules

- For each `(domainId, table)` pair you cover, emit **exactly one** entry.
- `methods` must be chosen from: `findById`, `list`, `insert`, `update`,
  `upsert`, `delete`, `softDelete` only.
- If the domain is `readOnly: true`, use only `findById` and/or `list`.
- `read`: use `{ "kind": "table", "name": "<table>" }` unless the product
  explicitly uses a view-backed read; then `kind` is `view` and `name` is the
  view name from `publicViews`.
- If `read.kind` is `view` and any write method is included, set `write` to
  `{ "kind": "table", "name": "<base_table>" }`.
- `upsert` requires `upsert.onConflict` (comma-separated column names as in
  Postgres).
- `softDelete` requires `softDelete.column` on the **write** table.
- `dto.include`: prefer `"all_columns"` unless a subset was explicitly
  requested; then `{ "columns": ["snake_case", ...] }`.
- `dto.style`: prefer `"zod"`.
- `idColumn` defaults to `"id"`; set explicitly if the primary key column has
  another name.
- Do not invent table or column names: every `read.name`, `write.name`, and DTO
  column must exist in the context and types.
- Optional `customMethods`: only when non-CRUD methods are explicitly requested;
  use `{ "name": "methodName" }` (camelCase).

## `meta.generator`

Use a truthful label (e.g. `cursor-agent`, `manual`).

## Output shape

```json
{
  "version": 1,
  "meta": {
    "generatedAt": "<ISO-8601>",
    "generator": "cursor-agent",
    "promptVersion": "v1"
  },
  "entries": []
}
```

Fill `entries` with one object per table. Each entry must include: `domainId`,
`table`, `read`, `methods`, `dto`.
