# Codegen Pipeline (Automated Backend Generation)

**Audience:** Developers consuming this template  
**Purpose:** Generate complete backend (repositories, actions, hooks, tests)
from database schema

---

## Overview

This template includes an **automated codegen pipeline** that reads your
Supabase database schema and generates:

- ✅ **Repositories** with DTOs, mappers, and ports
- ✅ **Server Actions** with Zod validation, auth, and logging
- ✅ **TanStack Query hooks (read-only)** with type-safe query keys
- ✅ **Unit & Integration Tests** for all generated code

> ⚠️ **Mutation hooks are not generated and do not exist.** All mutations go
> through Server Actions. See
> [data-access-pattern.md](../architecture/data-access-pattern.md).

**Zero manual coding required** - just run the pipeline!

## Pipeline Architecture

```

│ PHASE 0: LLM Semantic Analysis (ALL PLANS)                  │
│ Input:  packages/supabase-infra/src/types/database.types.ts │
│ Output:                                                     │
│   - config/domain-map.json (SSOT #1: domains, auth)         │
│   - config/repository-plan.json (SSOT #2: tables, methods)  │
│   - config/action-semantic-plan.json (SSOT #3: actions)     │
│ Command: pnpm codegen:actions-semantic-plan                 │
│ Automation: 100% LLM-powered (no human intervention)        │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ PHASE 1: Foundation (Repositories)                          │
│ Input:  config/repository-plan.json (SSOT #2)               │
│ Output:                                                     │
│   - modules/*/domain/dto/*.ts                               │
│   - modules/*/domain/ports/*.ts                             │
│   - modules/*/infrastructure/mappers/*.ts                   │
│   - modules/*/infrastructure/repositories/*.ts              │
│   - tests/integration/supabase-data/modules/*/*.test.ts     │
│ Command: pnpm codegen:backend --write                       │
│ Tests: Generated IMMEDIATELY after repositories             │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ PHASE 2: Business Logic (Actions)                           │
│ Input:  config/action-semantic-plan.json (SSOT #3)          │
│ Output:                                                     │
│   - actions/*/*.codegen.ts                                  │
│   - tests/unit/supabase-data/actions/*/*.test.ts            │
│ Command: pnpm codegen:actions-hooks --write                 │
│ Tests: Generated IMMEDIATELY after actions                  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ PHASE 3: UI Layer (Query Hooks — read-only)                 │
│ Input:  config/action-semantic-plan.json (SSOT #3)          │
│ Output:                                                     │
│   - hooks/*/*.codegen.ts  (query hooks only)                │
│   - tests/unit/supabase-data/hooks/*/*.test.ts              │
│ Command: pnpm codegen:actions-hooks --write                 │
│ Tests: Generated IMMEDIATELY after hooks                    │
│ NOTE: Mutation hooks are NOT generated.                     │
└─────────────────────────────────────────────────────────────┘
```

## Single Source of Truth (SSOT)

Each phase has **ONE source of truth** - no fallbacks, no dual sources:

| Plan                        | SSOT For                           | Generated By  | Consumed By                         |
| --------------------------- | ---------------------------------- | ------------- | ----------------------------------- |
| `domain-map.json`           | Domains, auth rules, codegen flags | Phase 0 (LLM) | Validation, organization            |
| `repository-plan.json`      | Tables, methods, DTOs              | Phase 0 (LLM) | Phase 1 (repositories)              |
| `action-semantic-plan.json` | Action semantics, inputs, outputs  | Phase 0 (LLM) | Phase 2 (actions) + Phase 3 (hooks) |

**Key Principle:** Each JSON file has a **clear, singular purpose**.

## Naming Convention

**ALL auto-generated files MUST follow this convention:**

```
*.codegen.ts          - Auto-generated TypeScript files
*.codegen.test.ts     - Auto-generated test files
*.codegen.test.tsx    - Auto-generated React test files
```

**Examples:**

- ✅ `packages/supabase-data/src/actions/patients/create-patient.codegen.ts`
- ✅
  `packages/supabase-data/src/hooks/patients/use-patient-query.hook.codegen.ts`
- ✅ `tests/unit/supabase-data/actions/patients/create-patient.codegen.test.ts`
- ❌ `packages/supabase-data/src/actions/create-patient.ts` (manual - no
  `.codegen.`)

**Why:** The clean script uses `.codegen.` to identify auto-generated files
safely.

## Commands

### Full Pipeline (ONE COMMAND - Recommended)

```bash
# Runs Phases 0-3 + validation automatically
pnpm codegen:full-pipeline
```

**What it does:**

1. ✅ Pre-flight checks (database.types.ts valid?)
2. ✅ Phase 0: Generate semantic plans
3. ✅ Phase 1: Generate repositories
4. ✅ Phase 2+3: Generate actions + hooks
5. ✅ Validation: typecheck, lint, tests
6. ✅ Auto-fix: Fixes common issues automatically

**Output:** Production-ready code, fully validated!

### Manual Phases (Advanced)

```bash
# Phase 0 only (generate plans)
pnpm codegen:actions-semantic-plan

# Phase 1 only (repositories)
pnpm codegen:backend --write
pnpm codegen:backend --check  # dry-run

# Phase 2 + 3 only (actions + hooks)
pnpm codegen:actions-hooks --write
pnpm codegen:actions-hooks --check  # dry-run
```

### Individual Phases

```bash
# Phase 0 only (generate plans)
pnpm codegen:actions-semantic-plan

# Phase 1 only (repositories)
pnpm codegen:backend --write
pnpm codegen:backend --check  # dry-run

# Phase 2 + 3 only (actions + hooks)
pnpm codegen:actions-hooks --write
pnpm codegen:actions-hooks --check  # dry-run
```

### Clean Auto-Generated Files

```bash
# Remove ALL auto-generated files (preserves manual files)
pnpm codegen:clean  # or: bash scripts/codegen/clean-autogenerated.sh
```

**What gets deleted:**

- ✅ All `*.codegen.ts` files in `packages/supabase-data/src/`
- ✅ All `*.codegen.test.ts` files in `tests/`

**What is PRESERVED:**

- ✅ `config/*.json` (plans)
- ✅ `packages/supabase-infra/src/types/database.types.ts`
- ✅ Manual actions (`_shared/`, `user-access/`, `user-roles/`, `profiles/`,
  `example/`)
- ✅ Manual tests (`tests/unit/codegen/`, `tests/integration/supabase-auth/`)

## Features

### Zod v4 Validation

All generated actions use **Zod v4** syntax:

```typescript
// ✅ CORRECT (Zod v4)
z.email()
z.uuid()
z.iso.datetime()
z.record(z.string(), z.unknown())
z.string().min(5, { error: "Too short" })

// ❌ WRONG (Zod v3 - deprecated)
z.email()
z.uuid()
z.record(z.unknown())
z.string().min(5, "Too short")
```

### Auth Patterns

All generated actions use centralized auth:

```typescript
const { userId, claims } = await requireAuth({
  action: "create_patient",
  requiredRole: "psychologist", // optional
})
```

### PHI Sanitization (HIPAA Compliance)

Error metadata is automatically sanitized:

```typescript
const AUDIT_SAFE_FIELDS = ["id", "psychologistId", "createdAt"] as const

await logServerEvent({
  // ...
  metadata: sanitizeForAudit(input, AUDIT_SAFE_FIELDS),
})
```

### Rate Limiting

Rate limiting is integrated into `requireAuth()`:

```typescript
// Called automatically inside requireAuth()
await checkActionRateLimit(userId) // 60 req/min per user
```

### Tenant Scoping

All tenant-scoped tables include `psychologist_id` filtering:

```typescript
const psychologistId = await getPsychologistIdForUser(userId)
const result = await repository.findByPsychologistId(psychologistId)
```

## Workflow

### Initial Setup (One-Time)

1. **Generate types from Supabase:**

   ```bash
   pnpm supabase:types:local  # or pnpm supabase:types:linked
   ```

2. **Run Phase 0 (generate plans):**

   ```bash
   pnpm codegen:actions-semantic-plan
   ```

3. **Review generated plans:**
   - `config/domain-map.json`
   - `config/repository-plan.json`
   - `config/action-semantic-plan.json`

4. **Run full pipeline:**

   ```bash
   pnpm codegen:backend --write
   pnpm codegen:actions-hooks --write
   ```

5. **Verify:**
   ```bash
   pnpm typecheck
   pnpm lint
   pnpm test
   ```

### After Schema Changes

1. **Regenerate types:**

   ```bash
   pnpm supabase:types:local
   ```

2. **Clean auto-generated files:**

   ```bash
   pnpm codegen:clean
   ```

3. **Re-run full pipeline:**
   ```bash
   pnpm codegen:actions-semantic-plan
   pnpm codegen:backend --write
   pnpm codegen:actions-hooks --write
   ```

## Manual vs Auto-Generated

### Manual Files (DO NOT REGENERATE)

These files are **manually maintained** and preserved by the clean script:

- `packages/supabase-data/src/actions/_shared/` - Shared utilities
- `packages/supabase-data/src/actions/user-access/` - Custom user access logic
- `packages/supabase-data/src/actions/user-roles/` - Role management
- `packages/supabase-data/src/actions/profiles/` - Profile management
- `packages/supabase-data/src/actions/example/` - Examples/placeholders

### Auto-Generated Files (SAFE TO DELETE)

These files are **regenerated** by the pipeline:

- All `*.codegen.ts` files
- All `*.codegen.test.ts` files
- All modules under `packages/supabase-data/src/modules/*/` (except manual ones)

## Troubleshooting

### "Cannot find module" errors

**Cause:** Imports missing `.codegen.` extension  
**Solution:** Regenerate with updated templates:

```bash
pnpm codegen:actions-hooks --write --force
```

### TypeScript errors in generated code

**Cause:** Database schema changed but types not regenerated  
**Solution:**

```bash
pnpm supabase:types:local
pnpm codegen:clean
pnpm codegen:actions-semantic-plan
pnpm codegen:backend --write
pnpm codegen:actions-hooks --write
```

### Missing repository exports

**Cause:** New modules not added to `package.json`  
**Solution:** Add exports to `packages/supabase-data/package.json`:

```json
{
  "exports": {
    "./modules/*": "./src/modules/*",
    "./actions/*": "./src/actions/*",
    "./hooks/*": "./src/hooks/*"
  }
}
```
