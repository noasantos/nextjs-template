# Migration Workflow Skill

## Purpose

Guide through migration creation following GR-015 (CLI Migrations Only).

## When to Trigger

Auto-trigger when user:

- Wants to create migration
- Mentions "database change" or "schema"
- Creates new table

## Skill Instructions

### **1. Mandatory Workflow**

```bash
# Step 1: Create migration via CLI
pnpm supabase:migration:new -- create_users_table

# Step 2: Develop in Supabase Studio SQL Editor
# Make your schema changes

# Step 3: Capture diff
pnpm supabase db diff -o supabase/migrations/YYYYMMDDHHMMSS_create_users_table.sql

# Step 4: Verify migration
pnpm supabase:migrations:verify

# Step 5: Test RLS
pnpm test:rls

# Step 6: Commit
git add supabase/migrations/YYYYMMDDHHMMSS_create_users_table.sql
git commit -m "feat: create users table"
```

### **2. Forbidden Patterns**

```bash
# ❌ FORBIDDEN - Manual file creation
touch supabase/migrations/manual.sql

# ❌ FORBIDDEN - Using db diff -f alone
pnpm supabase db diff -f create_users_table
```

### **3. Enforce Rules**

- ✅ **GR-015**: CLI-generated migrations only
- ✅ **Timestamp**: Generated by CLI
- ✅ **RLS Tests**: Mandatory for new tables
- ✅ **Verification**: Run migrations:verify

## Examples

### **User Request**

```
"I need to create a new users table"
```

### **Skill Response**

1. **RUN CLI command:**

   ```bash
   pnpm supabase:migration:new -- create_users_table
   ```

2. **DEVELOP in Studio:**
   - Open Supabase Studio
   - SQL Editor
   - Create table with RLS

3. **CAPTURE diff:**

   ```bash
   pnpm supabase db diff -o supabase/migrations/YYYYMMDDHHMMSS_create_users_table.sql
   ```

4. **CREATE RLS tests:**

   ```
   tests/rls/supabase-data/users.rls.test.ts
   ```

5. **RUN tests:**
   ```bash
   pnpm test:rls
   ```
